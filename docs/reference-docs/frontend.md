# Frontend Guidelines & Design

## 3.1 Design Philosophy

Analytical, not decorative

Minimal UI

Transparency-first

Separation of analysis and simulation

## 3.2 Navigation Structure
[ Overview | Data Explorer | Simulator | Insights ]

## 3.3 UI Guidelines

Neutral color palette

Consistent legends for risk levels

Clear labels for uncertainty

Avoid excessive animations

## 3.4 Simulator Page Design

Left Panel

Country selector

Commodity selector

Disruption type

Magnitude slider

Run Simulation button

Main Panel

Before vs After graph/map

Metric comparison table

Bottom Panel

LLM-generated explanation

Uncertainty and assumptions

## 3.5 Accessibility & Clarity

Tooltips for technical terms

Clear disclaimers on predictions

Readable charts and tables

## 3.6 Frontend Technology

Streamlit (recommended)

Optional: React (future scope)
